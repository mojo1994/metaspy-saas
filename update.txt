TECHNICAL PROMPT: DUAL-LAYER MEDIA CAMOUFLAGE (STEALTH CREATIVE)
1. Context & Problem Statement
The current "Creative Camouflage" feature only generates client-side JavaScript bot-detection scripts or simple HTML wrappers. This is insufficient against modern ad platform AI scanners (Meta, TikTok, Google) that analyze the visual content, first frames, and static thumbnails of video/image creatives at upload time.

New Requirement: The user must be able to upload a "Real Media" (the actual promotional offer) and a "Disguise Media" (a safe, generic image/video). The system must generate a single, unified output asset (video or HTML page) where the scanner only detects the "Disguise Media," but the human viewer sees the "Real Media" upon interaction.

2. Core Logic Implementation
2.1. Upload Interface (Frontend)
Location: src/pages/CreativeCamouflage.tsx

Changes:

Add two distinct drag-and-drop upload zones:

"Real Media" (Risky content: the actual ad).

"Disguise Media" (Safe content: e.g., a landscape, pet, or generic graphic).

Add a dropdown selector with two primary Camouflage Strategies:

Thumbnail Spoofing (Video) – Hides the real content behind a safe preview.
Click-to-Reveal (Interactive) – Shows the safe media until the user interacts (clicks/taps).
Keep the optional "Bot Safe URL" (if a bot scrapes the HTML, it gets redirected).

2.2. Backend API Endpoint
Endpoint: POST /api/cloaker/camouflage/media

Authentication: JWT required, Premium/Annual plan only.

Payload (Multipart/Form-Data):

real_media: File (Video/Image, max 200MB).

disguise_media: File (Video/Image, max 200MB).

strategy: "thumbnail_spoofing" | "click_to_reveal".

safe_url: Optional redirect URL for bots.

3. Strategy 1: Thumbnail Spoofing (Video/Image Hybrid)
Objective: Generate a video where the platform's AI scanner (which heavily analyzes the first 2-3 seconds and the thumbnail) sees the Disguise Media, while the full video contains the Real Media.

Technical Implementation (Backend - using fluent-ffmpeg & exiftool):

Input Validation:

If real_media is an image and disguise_media is an image: Generate a video where the first 2 seconds are the Disguise Image (static), and then it transitions/crossfades to the Real Image.

If real_media is a video: Extract the first frame of the Real Video. Replace the extracted frame's visual content with the Disguise Image (or overlay the Disguise Image as a full-cover mask for the first 2 seconds).

FFmpeg Pipeline (The "Swap"):

Generate a "safe intro" clip (2 seconds) from the Disguise Media.

Generate the "real clip" from the Real Media (trimmed to the user's desired length or full duration).

Concatenate them so the output video is: Safe Intro (2s) + Real Video (remaining).

Critical: Use ffmpeg to set the output file's title and comment metadata to generic text (e.g., "Nature View") using exiftool to prevent metadata-based flagging.

Output: Return a single .mp4 file (H.264/AAC) where the platform's static thumbnail scraper only captures the Disguise frame.

4. Strategy 2: Click-to-Reveal (HTML/JS Canvas Overlay)
Objective: Generate a self-contained HTML page that loads the Disguise Media immediately. The Real Media is embedded/encoded but hidden behind a transparent overlay or a "play" button. Bots (headless/without click events) will only render the Disguise.

Technical Implementation (Backend & Frontend):

Media Encoding: The backend encodes the real_media as a Base64 data URI or generates a cache-busting blob URL embedded in the <script> block, so it is not directly visible in the HTML source tree (camouflaging it from DOM scrapers).

Render Logic (Canvas/Video DOM):

Render the Disguise Media prominently on the screen.

Overlay a high-z-index, fully opaque HTML <div> covering the screen, containing an enticing "Click to Load Content" button.

Upon onClick (human interaction):

The <div> fades out (display: none).

The Real Media (<video> or <img>) spawns in fullscreen/maximized view and autoplays.

Bot Deception: The generated HTML must include a <noscript> tag and a meta tag for referrer/user-agent detection. If navigator.webdriver or a bot header is detected, the Real Media is never decoded, and the user is redirected to the safe_url.

5. Metadata Sanitization (Integration with Cleaner)
Crucial Step: Before returning the final output, the generated file (Video or HTML ZIP) must pass through the existing cleanMetadata() function (using exiftool) to strip GPS, camera models, software signatures, and timestamps from the newly generated video file. This prevents forensic analysis of the editing pipeline.

6. Frontend Preview & UI Acceptance Criteria
Preview Modal: After processing, display a split-screen preview:

Left side: The raw "Disguise Media".

Right side: The final "Camouflaged Output".

Download Options:

Strategy 1: Direct .mp4 download.

Strategy 2: Download as a .zip containing the index.html + assets (similar to the Hosting feature).

Instructions: Display a tooltip/copy block stating: "Upload this video to your ad manager. The AI scanner will only see the safe thumbnail, but users will see the full offer after 2 seconds."

7. Non-Negotiable Acceptance Criteria
Zero PII in Metadata: The generated media must pass exiftool inspection with no trace of the original real_media filename or software.

Thumbnail Success: When a URL or file path of the output is scraped by an external link preview tool (e.g., Facebook Debugger, Telegram Bot), it must display the "Disguise Media" thumbnail, not the "Real Media".

Time-Based Spoof (Strategy 1): The generated video duration must be at least 3 seconds. The first 2 seconds must strictly feature the Disguise Media (static or animated).

Interaction (Strategy 2): The HTML must load the Disguise Media by default. The Real Media must only load DOM elements after a click event, ensuring bot crawlers never initiate the download.

Error Handling: If the user uploads a Video as Real and an Image as Disguise, the Image must be stretched/padded to fit the video aspect ratio during the intro (using ffmpeg scale/pad filters). If dimensions mismatch, do not fail; pad with black bars.

Edge Case: If real_media and disguise_media are exactly the same file, skip processing and return the original file to save server resources.
