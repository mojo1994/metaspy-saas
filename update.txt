# CRITICAL UI/UX OVERHAUL — METASPY SALES PAGE (HERO, PRICING, COMPARISON TABLE)

## CONTEXT
This is a SaaS sales/landing page (MetaSpy — Meta Ads intelligence tool) currently generated by an AI builder. The page works functionally but looks like a generic, lifeless AI-template output: static, flat, with broken layout structure on the comparison table, duplicated/generic icons, and zero motion anywhere on the page. Your job is to rebuild the visual and interaction layer so this page looks like a premium, hand-crafted product page that would belong next to Linear, Stripe, or Vercel's marketing site — something that makes a visitor stop scrolling and think "this is a serious, well-funded product."

## STEP 0 — DETECT STACK BEFORE TOUCHING ANYTHING
Before writing any code, inspect the existing codebase and identify: framework (plain HTML/CSS/JS, React, Vue, or AI-builder-specific structure), how styles are organized (inline, CSS file, Tailwind, styled-components), and how the pricing/comparison data is currently rendered (hardcoded HTML or mapped from a data array). Adapt your implementation to fit the detected stack — do not introduce a new framework or rewrite the entire app shell. Fix and modernize in place.

## CRITICAL CONSTRAINT
**DO NOT change the existing color palette.** Keep the dark background, the purple/violet primary accent, and existing neutral tones exactly as they are. All improvements below must be achieved through layout, typography, iconography, spacing, and motion — never through new colors.

## PROBLEM 1 — ICON SYSTEM IS BROKEN (DUPLICATED, GENERIC, INCONSISTENT)
Current state: the hero feature cards (MetaSpy Ad Intelligence, Clonador, Cloacker, Análise Avançada, Filtros Inteligentes, Exportação CSV) use a mix of generic/duplicated icons — the same circular "target" icon appears on multiple unrelated cards, a heart icon is used for "Clonador" (no semantic relationship), and icon visual weight is inconsistent (some bold, some thin).

**Fix:**
- Design or source ONE consistent SVG icon set, outline-style, 1.5–2px stroke, identical viewBox/bounding box (e.g., 24x24) across every icon, all inheriting `currentColor` so they pick up the existing purple accent automatically.
- Assign icons by actual semantic meaning, no repeats:
  - MetaSpy Ad Intelligence → radar/scan icon (sweeping radar sweep, not a static bullseye)
  - Clonador → a "copy/duplicate" or "stacked layers" icon, NOT a heart
  - Cloacker → a shield or eye-slash (stealth/protection) icon
  - Análise Avançada → a bar-chart/analytics icon
  - Filtros Inteligentes → a sliders/funnel icon
  - Exportação CSV → a download/export-to-file icon
- No two cards may share the same icon under any circumstance.
- Icons must sit inside a soft rounded container with a subtle inset glow (using existing purple at low opacity) — and that container must animate (scale + glow intensify) on card hover, not stay static.

## PROBLEM 2 — COMPARISON TABLE LAYOUT IS STRUCTURALLY BROKEN
Current state: on `/planos`, the "Comparativo Completo" table renders Basico / Gold / Premium as misaligned, wrapped elements — "Premium" drops to its own line below the header row, and each feature row shows its three checkmarks stacked vertically (one centered below the other) instead of in three clean side-by-side columns. This looks like a broken CSS grid, not a real comparison table.

**Fix:**
- Rebuild as a proper 4-column grid/table: Column 1 = feature name (left-aligned), Columns 2–4 = Básico / Gold / Premium, each perfectly vertically aligned in their own column for every single row.
- Header row must have all three plan names in one single row, same baseline, with "Gold" visually emphasized (border, background tint using existing purple, "MELHOR VALOR" badge) since it's the recommended tier.
- Checkmarks (✓) and dashes (–) must be horizontally centered within their column and vertically centered within their row — no row should ever wrap or stack.
- Add zebra-striping or alternating subtle row background (using existing dark neutrals at different opacity) to improve scanability across 8+ rows.
- On scroll into view, animate rows in with a staggered fade+slide-up (60-80ms stagger per row) — the table must not just "appear" instantly.
- Make the table horizontally scrollable with momentum on mobile/narrow viewports instead of breaking into stacked chaos.

## PROBLEM 3 — TYPOGRAPHY IS FLAT AND UNDIFFERENTIATED
Current state: headline, subheadline, card titles, and body text all use the same static system font with minimal weight/size contrast — nothing visually signals what's most important on the page.

**Fix:**
- Import a distinctive display font for headlines (e.g., a geometric sans like "Geist", "Inter Display", or "General Sans" at 700–800 weight) paired with a clean readable body font (Inter or similar at 400–500 weight). Self-host or load via efficient `font-display: swap`.
- Establish a strict type scale: Hero H1 (e.g., 64-72px desktop), Section H2, Card title, Body, Caption/meta — each with deliberate size/weight/line-height steps, not arbitrary px values.
- The current gradient-text headline ("Inteligência de Ofertas em Escala") should use a smoother, more controlled gradient (existing purple tones only) with proper `background-clip: text` rendering — currently it reads as slightly muddy/low contrast against the dark background.
- Price numbers in the pricing cards (R$ 49,90 / R$ 97,00 / R$ 197,00) need stronger visual dominance — increase weight contrast against the "/mes" suffix and the strikethrough original price, which should be visually deprioritized (smaller, muted, with a clean diagonal strike, not just text-decoration default).

## PROBLEM 4 — ZERO MOTION ANYWHERE ON THE PAGE
Current state: the entire page is dead static. Cards, buttons, badges, and the table all appear instantly with no transitions, no hover feedback beyond default browser behavior, no scroll-triggered reveals.

**Fix — implement a full motion system across the whole page:**
- **Hero entrance**: headline, subheadline, and CTA buttons fade up sequentially (120ms stagger) on page load using `cubic-bezier(0.16, 1, 0.3, 1)` easing — never instant render.
- **Feature cards (hero grid)**: animate in with scroll-triggered staggered reveal (IntersectionObserver or framework-native scroll trigger), each card lifting slightly (translateY 16px → 0) with fade. On hover: card lifts (translateY -4px), border glow intensifies, icon container scales 1.05–1.1 with smoothed easing — all in one cohesive `transform` + `box-shadow` transition, 200-300ms.
- **Pricing cards**: the "Gold" (highlighted/best value) card should have a continuous, subtle ambient glow/pulse animation on its border (slow, 3-4s loop, low-amplitude) to draw the eye — but must respect `prefers-reduced-motion`.
- **"MELHOR VALOR" badge**: should have a subtle shimmer or gentle pulse, not be a static flat label.
- **Buttons**: "Assinar Agora" / "Começar Agora" need press feedback (scale 0.97 active state) and a hover state where a subtle light sweep or background shift animates across the button (200ms), not just a flat color change.
- **CTA buttons in header** (Entrar / Criar Conta): smooth hover transitions, no instant color snap.
- **Page-to-page transition** (Hero → /planos): if feasible within the stack, add a subtle fade/slide transition between pages rather than a hard reload-style cut.
- **Scroll-linked reveals**: every major section (hero, feature grid, pricing, comparison table) must use scroll-triggered entrance animation — nothing below the fold should "already be there" when scrolled into view; everything earns its appearance.
- Animate only `transform`, `opacity`, `filter`, and `box-shadow` (via layered shadow swap, not literal blur animation) to avoid layout jank. Add `will-change` judiciously on actively-animating elements only.

## PROBLEM 5 — GENERIC AI-BUILDER SPACING AND CARD STRUCTURE
