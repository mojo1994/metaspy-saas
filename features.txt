3.2 Cloacker Feature (Creative Cloaker)
Purpose: Provide a script generator that produces a JavaScript snippet to cloak a webpage (i.e., show different content to bots vs. humans).

Requirements:

Users can input a target URL (the page to cloak) and a safe page URL (for bots).

The system generates a self-contained HTML/JS script that includes:

User-agent detection, IP reputation checking, browser fingerprinting, etc.

Logic to redirect or render different content.

The generated script should be downloadable or provided as a code snippet.

Backend logic:

Endpoint to generate cloaker script (POST).

Store generated scripts per user (for history).

Script parameters: target URL, safe URL, optional custom rules.

Security: Sanitize inputs, prevent code injection, rate-limit generation.

3.3 Subscription & Payment Integration (Kirvano)
Goal: Replace the free limit with a monthly subscription plan (only monthly, no annual). Free tier remains but with stricter limits (e.g., 3 clones total, no cloaker access). Premium monthly unlocks unlimited clones, cloaker usage, and advanced options.

Kirvano Integration:

Use Kirvano’s Webhook API to handle payment confirmations.

Endpoints required:

POST /api/subscription/webhook – to receive Kirvano events (payment.success, subscription.canceled, etc.).

POST /api/subscription/create-checkout – returns a checkout URL to redirect user.

GET /api/subscription/status – returns current user’s subscription status (active/inactive).

Subscription Flow:

User clicks “Upgrade” in UI.

Backend creates a Kirvano checkout session (monthly plan) and returns URL.

User completes payment on Kirvano.

Kirvano sends webhook to our backend.

Backend updates user’s subscriptionStatus to active and sets expiryDate (30 days from payment).

On each API request, check subscription status and enforce limits.

If subscription expires, user reverts to free tier; webhook handles cancellations.

Database Changes:

Add fields to User model:

subscriptionStatus: enum['free', 'active', 'cancelled', 'expired']

subscriptionId: string (Kirvano transaction ID)

subscriptionExpiry: Date

clonesUsed: number (reset monthly? decide: clones are total used – we may change to monthly quota for premium)

cloakerScripts: array (to store generated scripts metadata)

Endpoint Summary (new):

POST /api/subscription/create-checkout – body: { plan: 'monthly' } (only monthly allowed).

POST /api/subscription/webhook – Kirvano webhook handler (verify signature).

GET /api/subscription/status – returns current subscription info.

Request: { targetUrl, safeUrl, customRules?: object }

Validate URLs, sanitize.

Generate a unique script ID.

Return: { scriptId, scriptCode, downloadLink }

4.5 Security Enhancements
Input Validation: All endpoints must validate and sanitize inputs (URLs, strings).

Authentication: JWT tokens with short expiry (15 min) + refresh token.

CORS: Restrict to frontend domain.

Rate Limiting:

Clone endpoint: 5 requests per minute for free, 20 for premium.

Cloaker generate: 10 requests per hour for premium, disabled for free.

Webhook Security: Verify Kirvano webhook signatures using secret key (provided by Kirvano).

Data Encryption: Sensitive data (e.g., user passwords) hashed with bcrypt.

Logging & Monitoring: Log all administrative actions (subscription changes, deletions).

GDPR Compliance: Allow user data export and deletion.

5. API ENDPOINT SPECIFICATIONS (New & Modified)
5.1 Subscription Endpoints
POST /api/subscription/create-checkout
Auth: Required (JWT).
Request:

json
{ "plan": "monthly" }
Response:

json
{ "checkoutUrl": "https://kirvano.com/checkout/..." }
Logic:

Validate plan (only 'monthly').

Call Kirvano API to create a checkout session with success/cancel URLs.

Return the URL.

POST /api/subscription/webhook
Auth: None (but verify signature).
Request: Kirvano payload (JSON).
Response: 200 OK.
Logic:

Verify signature using stored secret.

Parse event type (payment.success, subscription.cancelled, etc.).

Update user’s subscription status and expiry accordingly.

If payment.success: set status to 'active', expiry = now + 30 days.

If subscription.cancelled: set status to 'cancelled' (still active until expiry).

GET /api/subscription/status
Auth: Required.
Response:

json
{
  "status": "active",
  "expiry": "2026-07-21T...",
  "plan": "monthly",
  "features": { "unlimitedClones": true, "cloaker": true }
}
5.2 Cloaker Endpoints
POST /api/cloaker/generate
Auth: Required, premium only.
Request:

json
{
  "targetUrl": "https://example.com/offer",
  "safeUrl": "https://example.com/safe",
  "customRules": { "userAgents": ["Googlebot"], "ipBlocks": ["192.168.0.0/16"] } // optional
}
Response:

json
{
  "scriptId": "uuid",
  "scriptCode": "// generated JS code",
  "downloadLink": "/api/cloaker/script/uuid/download"
}
Logic:

Validate URLs (must be valid HTTP/HTTPS).

Generate a unique script using a template (e.g., based on fingerprintjs, user-agent parsing).

Store in DB with user ID.

Return the script.

GET /api/cloaker/scripts
Auth: Required.
Response: List of scripts with metadata (id, targetUrl, createdAt, etc.).

GET /api/cloaker/script/:id
Auth: Required, ownership check.
Response: The script code and metadata.

DELETE /api/cloaker/script/:id
Auth: Required, ownership check.
Response: 204 No Content.
