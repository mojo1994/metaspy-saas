# MetaSpy SaaS - Resumo do Projeto

## Estrutura
- `/server/index.js` → todas as rotas Express (auth, email, subscription, pages, builder, public hosting, camouflage, cleaner)
- `/server/email.js` → templates HTML + envio Maileroo
- `/server/db.js` → pool PostgreSQL + schema init (7 tabelas: users, pages, cloaker_scripts, cloak_detections, camouflage_scripts, cleaned_assets, email_codes)
- `/server/cloudflareStorage.js` → upload/download/delete R2 via S3 API + getWorkerUrl()
- `/src/pages/` → frontend React
- `/src/components/builder/` → visual Page Builder (drag-and-drop)
- `/netlify.toml` + `/public/_redirects` → redirects: `/api/*` e `/p/*` → Render
- `/cloudflare-worker.js` + `/wrangler.toml` → Worker code + config

## O que já foi implementado

### 1. Autenticação
- Signup com verificação de email (código de 6 dígitos via Maileroo)
- Login com JWT + refresh token
- Recuperação de senha (código por email)
- Middleware `authMiddleware` que decodifica JWT e anexa `req.user`

### 2. Planos e Pagamentos (Kirvano)
- Tabela `users` com `plan` (nenhum/mensal/anual), `subscription_expiry`, `email_verified`
- Checkout: links estáticos Kirvano por plano
- Webhook Kirvano: ativa/desativa planos, envia email de confirmação
- Rate limiting, admin middleware (email específico)

### 3. Email (Maileroo)
- Templates: verificação, recuperação, confirmação de compra, pending checkout
- Todos templates incluem: "⚠ Não encontrou? Verifique sua caixa de spam."
- Endpoint `POST /api/send-email` para disparos programáticos

### 4. Páginas / Hospedagem
- **Paginas** (`DashboardPages.tsx` + `PageEditor.tsx`): criar/editar HTML puro com textarea + preview ao vivo
- **Hospedar** (`HostPage.tsx`): upload de pasta `.zip` com arrastar-e-soltar (200MB, requer `index.html`)
- Upload ZIP extrai `index.html` para a coluna `html` do banco (não depende de disco)
- Servidas em `centralspyads.netlify.app/p/{slug}` via redirect do Netlify → Render → coluna `html`
- Tipos: `'page'` (HTML colado), `'hosted'` (upload ZIP), `'builder'` (árvore do Page Builder)

### 5. Visual Page Builder (Criar Pagina)
- Página: `PageBuilder.tsx`, rota `/dashboard/hospedar/criar`
- Modelo de documento: `documentModel.ts` (23 tipos de nó, sistema de estilos, CSS generator)
- Painel esquerdo: árvore de elementos (`ElementTree.tsx`) + paleta de widgets
- Canvas central (`BuilderCanvas.tsx`): drag-and-drop, preview responsivo, zoom, drop zones
- Painel direito (`PropertyInspector.tsx`): editor contextual (conteúdo, layout, aparência, tipografia, interações, animação)
- **6 features implementadas:**
  1. Salvar no backend (API + auto-save 30s)
  2. Carregar do backend (`?edit={id}`)
  3. Componentes reutilizáveis (localStorage)
  4. Upload de imagem (multer + R2)
  5. Animações scroll + hover/interações click
  6. Publicar como página hospedada (gera HTML, salva no banco, disco + R2)

### 6. Ferramentas
- Metadata Cleaner (remover metadados de fotos com exiftool)
- Cloaker: gerador de script JS + detector multi-user-agent + camuflagem de criativos
- Page Vault Tool
- MetaSpy Tool (gerador de pixel)
- Deep Clone: download de site completo + tree viewer + ZIP

### 7. Infra
- Render: PostgreSQL, `ssl: rejectUnauthorized: false`, `app.set('trust proxy', 1)`
- Netlify: `_redirects` + `netlify.toml`, proxy `/api/*` e `/p/*` → Render
- Cloudflare Worker (metaspy-host): lê do R2, não usado no caminho crítico (fallback)
- Render free tier dorme após 15min (cold start ~30-60s)
- Auto-deploy: push na main → Render + Netlify

## Schema do Banco (tabela `pages`)
```sql
CREATE TABLE pages (
  id TEXT PRIMARY KEY, user_id TEXT NOT NULL, slug TEXT NOT NULL UNIQUE,
  title TEXT NOT NULL DEFAULT 'Sem titulo', html TEXT NOT NULL DEFAULT '',
  type TEXT NOT NULL DEFAULT 'page', published INTEGER NOT NULL DEFAULT 0,
  cf_url TEXT, created_at TEXT NOT NULL, updated_at TEXT NOT NULL
);
```
